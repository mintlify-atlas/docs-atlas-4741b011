## View User Details

<RequestExample>
```bash Request
curl -X GET https://api.beanquick.com/api/admin/usuarios/1 \
  -H "Authorization: Bearer {token}" \
  -H "Content-Type: application/json"
```
</RequestExample>

<ResponseExample>
```json Response
{
  "id": 1,
  "name": "John Doe",
  "email": "john@example.com",
  "rol": "cliente",
  "created_at": "2026-01-15T10:30:00.000000Z",
  "updated_at": "2026-01-15T10:30:00.000000Z"
}
```
</ResponseExample>

<ParamField path="id" type="integer" required>
  The ID of the user to retrieve
</ParamField>

<ResponseField name="id" type="integer">
  User ID
</ResponseField>

<ResponseField name="name" type="string">
  User's full name
</ResponseField>

<ResponseField name="email" type="string">
  User's email address
</ResponseField>

<ResponseField name="rol" type="string">
  User role: `admin` or `cliente`
</ResponseField>

<ResponseField name="created_at" type="string">
  Account creation timestamp
</ResponseField>

<ResponseField name="updated_at" type="string">
  Last update timestamp
</ResponseField>

## Update User

<RequestExample>
```bash Request
curl -X PUT https://api.beanquick.com/api/admin/usuarios/1 \
  -H "Authorization: Bearer {token}" \
  -H "Content-Type: application/json" \
  -d '{
    "name": "John Smith",
    "email": "johnsmith@example.com",
    "rol": "admin",
    "password": "newSecurePassword123"
  }'
```
</RequestExample>

<ResponseExample>
```json Response
{
  "message": "Usuario actualizado correctamente.",
  "usuario": {
    "id": 1,
    "name": "John Smith",
    "email": "johnsmith@example.com",
    "rol": "admin",
    "created_at": "2026-01-15T10:30:00.000000Z",
    "updated_at": "2026-03-05T11:45:00.000000Z"
  }
}
```
</ResponseExample>

<ParamField path="id" type="integer" required>
  The ID of the user to update
</ParamField>

<ParamField body="name" type="string" required>
  User's full name (max 255 characters)
</ParamField>

<ParamField body="email" type="string" required>
  User's email address (must be unique)
</ParamField>

<ParamField body="rol" type="string" required>
  User role: `admin` or `cliente`
</ParamField>

<ParamField body="password" type="string">
  New password (minimum 6 characters). Optional - only include to change password
</ParamField>

<ResponseField name="message" type="string">
  Success message
</ResponseField>

<ResponseField name="usuario" type="object">
  Updated user object
  <Expandable title="properties">
    <ResponseField name="id" type="integer">
      User ID
    </ResponseField>
    <ResponseField name="name" type="string">
      Updated name
    </ResponseField>
    <ResponseField name="email" type="string">
      Updated email
    </ResponseField>
    <ResponseField name="rol" type="string">
      Updated role
    </ResponseField>
    <ResponseField name="updated_at" type="string">
      Timestamp of the update
    </ResponseField>
  </Expandable>
</ResponseField>

<Note>
  The password field is optional. If provided, it must be at least 6 characters long and will be encrypted before storage. The email must be unique across all users except the current user being updated.
</Note>

## Delete User

<RequestExample>
```bash Request
curl -X DELETE https://api.beanquick.com/api/admin/usuarios/1 \
  -H "Authorization: Bearer {token}" \
  -H "Content-Type: application/json"
```
</RequestExample>

<ResponseExample>
```json Success Response
{
  "message": "Usuario eliminado correctamente."
}
```

```json Self-Deletion Error
{
  "message": "No puedes eliminar tu propia cuenta de administrador."
}
```

```json Error Response
{
  "message": "Error al eliminar el usuario.",
  "error": "Database constraint violation"
}
```
</ResponseExample>

<ParamField path="id" type="integer" required>
  The ID of the user to delete
</ParamField>

<ResponseField name="message" type="string">
  Success or error message
</ResponseField>

<ResponseField name="error" type="string">
  Detailed error information (only present on error)
</ResponseField>

<Warning>
  Administrators cannot delete their own account. This is a security measure to prevent accidental self-deletion. If an admin attempts to delete themselves, a 403 Forbidden response will be returned.
</Warning>

<Note>
  This operation permanently deletes the user account. Ensure you have proper backups and that the deletion is intentional.
</Note>
